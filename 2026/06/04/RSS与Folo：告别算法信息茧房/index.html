RSS 与 Folo：告别算法，主动掌控你的信息流

Posted on 2026-06-04 In tech
RSS 是一种古老而优雅的信息订阅协议，配合现代阅读器 Folo，可以帮助我们从算法推荐的信息茧房中解脱出来，主动掌控自己的信息流。

我们每天打开微博、抖音、小红书，刷到的内容是算法帮你"精心挑选"的——但这真的是你想要的信息吗？本文聊聊一种更古老、更纯粹的信息获取方式：RSS，以及如何用现代工具 Folo 把它发挥到极致。

什么是 RSS？

RSS（Really Simple Syndication，简易信息聚合）是一种基于 XML 的 Web 内容分发格式，诞生于 1999 年。它的核心思想非常简单：网站将自己的更新内容以标准格式发布，读者用 RSS 阅读器订阅，就能自动接收所有新内容，完全不依赖算法推荐。

想象一下：你订阅了 10 个博客、5 个科技媒体、3 个 GitHub Release 通知，所有新内容都按时间顺序整齐地出现在你的阅读器里——没有广告插入，没有算法干预，没有"你可能喜欢"。这就是 RSS 的魅力所在。

一个典型的 RSS Feed URL 长这样：

https://sspai.com/feed
https://github.com/vitejs/vite/releases.atom
https://feeds.feedburner.com/hacker-news/newest

算法推荐与信息茧房

现代平台的推荐算法本质上是一个商业工具，而非信息工具。它的目标是最大化你的停留时长和互动率，而不是帮你构建完整的知识体系。算法会不断强化你已有的偏好，过滤掉"不够吸引你"的内容，久而久之便形成了所谓的"信息茧房"——你只看到了你"爱看"的，而不是你"该看"的。

更危险的是，你很难意识到自己身处茧房之中。你觉得自己每天都在"获取信息"，实际上只是在消费经过筛选的娱乐内容。对于想要保持独立思考、持续学习的人来说，RSS 订阅是一种很好的解药：你主动选择信息源，算法无从插手。

RSS 的工作原理

RSS 的技术实现相当简洁。网站服务器维护一个 .xml 或 .atom 格式的文件，每次发布新内容时更新该文件。RSS 阅读器定期轮询（Poll）这些 URL，检测到更新后便拉取新条目显示给用户。

整个流程可以概括为：

  1. 用户在阅读器中添加一个 Feed URL（如 https://example.com/feed.xml）
  2. 阅读器按设定间隔（如每 15 分钟）请求该 URL
  3. 服务器返回 XML 文档，包含最新文章的标题、摘要、链接、发布时间等
  4. 阅读器解析 XML，将新条目呈现给用户
  5. 用户点击感兴趣的条目，跳转到原文

这套机制完全去中心化，不依赖任何第三方平台，也不需要账号。网站只要提供 Feed，任何人都能订阅。

认识 Folo

Folo 是一款现代化的 RSS 阅读器，由 RSSNext 团队开发，开源且跨平台（支持 macOS、Windows、Linux、iOS、Android 以及 Web 端）。与传统 RSS 阅读器相比，Folo 有几个显著亮点：

  • AI 摘要与翻译：对订阅内容自动生成摘要，支持多语言翻译，可以快速判断是否值得精读
  • 内容聚合增强：不只支持标准 RSS/Atom，还支持订阅 Twitter/X 用户、YouTube 频道、播客、GitHub Release 等多种内容类型
  • Power（积分）系统：Folo 内置了一套基于区块链的创作者激励机制，用户可以用 Power 打赏优质内容创作者
  • 极简 UI：界面设计非常克制，以内容为中心，减少视觉干扰
  • 跨设备同步：登录账号后，已读状态、收藏、订阅列表全平台同步

如何使用 Folo 优化信息检索

下面是我个人使用 Folo 构建信息流的方法，供大家参考：

第一步：按主题建立分组（Lists）

不要把所有订阅混在一起，按主题分组管理。例如我会分为：Tech News、AI/ML、Finance、Indie Blogs、GitHub Updates 等。这样打开阅读器时可以按优先级快速浏览不同分组。

第二步：设置阅读节奏

RSS 的优势是异步获取，不必实时盯着。建议设定固定的"阅读时间"，比如每天早上花 20 分钟浏览昨天的更新，而不是随时随地刷。这能大幅降低信息焦虑。

第三步：利用 AI 摘要快速筛选

Folo 的 AI 摘要功能可以帮你快速判断哪些文章值得深读，哪些可以跳过。对于英文内容，AI 翻译功能也能节省大量时间。遇到真正感兴趣的内容，再点进去精读原文。

第四步：订阅 GitHub Release 追踪技术动态

对于开发者来说，Folo 订阅 GitHub Release 是个非常实用的功能。比如订阅 https://github.com/vitejs/vite/releases.atom，就能第一时间知道 Vite 发布了新版本，而不需要去刷 Twitter 或等别人转发。

第五步：找到并收藏高质量 Feed

Folo 有一个社区共享的订阅源发现功能，可以看到其他用户订阅了哪些 Feed，方便发现自己不知道的优质内容源。这是一种比算法推荐更可信的"社交推荐"，因为背后是真实用户的主动选择。

推荐订阅源

这里分享几个我个人在用的订阅源，涵盖技术、AI 和独立博客方向：

  • Hacker News 热帖：https://hnrss.org/frontpage — 每天技术圈最热的讨论，纯社区投票，没有算法干预
  • 少数派：https://sspai.com/feed — 高质量中文数字生活和效率工具内容
  • Paul Graham（pg）：http://www.paulgraham.com/rss.html — YC 创始人，关于创业和人生的深度文章
  • The Batch（吴恩达）：https://www.deeplearning.ai/the-batch/feed/ — 每周 AI 领域最值得关注的进展，干货密度极高
  • GitHub Trending（非官方）：https://rsshub.app/github/trending/daily/any/zh — 每日 GitHub 热门项目，通过 RSSHub 实现
  • arxiv cs.AI：https://rss.arxiv.org/rss/cs.AI — AI 领域最新论文，适合跟踪学术前沿

其中 RSSHub 值得特别一提，它是一个开源项目，可以为几乎任何没有原生 RSS 的网站生成订阅源，包括微博、B 站、知乎等，大大扩展了 RSS 的覆盖范围。

总结

信息时代，选择"如何获取信息"本身就是一种重要的能力。算法推荐降低了信息获取的门槛，但也在悄悄削弱你的信息自主权。RSS 让你重新成为信息流的主人——你决定关注谁，你决定什么时候读，你决定读多久。

Folo 把这套古老的协议包装进了现代化的体验，AI 摘要、多内容类型支持、跨平台同步，让 RSS 不再是极客的专利，而是任何人都能上手的工具。如果你也厌倦了被算法牵着鼻子走，不妨从今天开始，花半小时搭建属于自己的信息订阅体系。

我的 Folo 订阅列表还在持续迭代中，欢迎在评论区交流你的 RSS 使用心得 :)
